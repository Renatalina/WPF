Frame - класс
 
Рамка является элементом управления содержимым, который поддерживает переходы.

Пространство имен:  System.Windows.Controls

Использование элемента объекта XAML
<Frame>
  Content
</Frame>

Конструкторы

 Frame	Инициализирует новый экземпляр Frame класс.

Заметки

Frame элемент управления содержимым, который обеспечивает возможность перехода и отобразить содержимое. Frame можно размещать в другое содержимое, как и в случае с другими элементами управления и элементами.
 
 Внимание
После Frame элемент управления выполняет переход к содержимому HTML Frame элемент управления автоматически создает собственный элемент управления ActiveX WebBrowser. WPF включает средства безопасности путем применения функции управления к элементу управления ActiveX WebBrowser. Элементы управления функциями, которые применяются для Xbap отличается и автономных приложений. Некоторые приложения должны применяться дополнительные элементы управления функциями для предотвращения вредоносного содержимого. Дополнительные сведения см. в разделе "управление и функция WebBrowser управляют" раздел Безопасность (WPF) и Обзоры и учебники элемента управления WebBrowser.

Содержимое может быть любым типом .NET Framework объект и HTML файлы. Как правило, однако страницы предпочитаемые переходу к содержимому пакета для навигации (см. Page).
Содержимое может переходить к установкой Source свойство с URI для нужного содержимого. Кроме того, можно переходить к содержимому с помощью одной из перегрузок следующих Navigate метод:
Navigate(Uri)
Navigate(Uri, Object)
Переход к которым следуют когда содержимое URI" Frame возвращает объект, содержащий содержимое. Кроме того, можно переходить к содержимому с помощью одного из Navigate перегруженного варианта метода, который принимает объект.
Navigate(Object)
Navigate(Object, Object)
